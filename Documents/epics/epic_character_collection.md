# Epic: キャラクター収集と進化

## 概要
攻略の成果を可視化し、継続的なモチベーションを生み出すための「報酬システム」。

## ユーザーストーリー

### 1. クエスト達成報酬の獲得
*   **As a (誰):** 子供
*   **I want to (何):** 読み攻略（10問クリアなど）を達成した直後に、宝箱が開く演出と共に新しいキャラクターをゲットしたい。
*   **So that (なぜ):** 「頑張ってクリアした甲斐があった！」という達成感を即座に味わい、次のプレイへの意欲を高めたいから。

### 2. 図鑑でのコレクション確認
*   **As a (誰):** 子供
*   **I want to (何):** 今まで集めたキャラクターを「図鑑」で一覧表示し、それぞれの名前や姿をじっくり眺めたい。
*   **So that (なぜ):** 自分の冒険の成果（資産）を目に見える形で確認し、コンプリート欲求を満たしたいから。

### 3. 未取得キャラのチラ見せ
*   **As a (誰):** 子供
*   **I want to (何):** まだゲットしていないキャラクターを図鑑で「シルエット」や「？」マークとして確認したい。
*   **So that (なぜ):** 「次はどんなキャラなんだろう？」「早くこのシルエットを埋めたい！」というワクワク感（期待感）を持ち続けたいから。

### 4. キャラクターの進化
*   **As a (誰):** 子供
*   **I want to (何):** 特定の条件（レベルアップや特定アイテム使用）を満たして、キャラクターをより豪華で強力な姿に「進化」させたい。
*   **So that (なぜ):** 自分が手塩にかけて育てた相棒が強くなる姿を見て、大きな驚きと達成感を得たいから。

## 完了基準 (Acceptance Criteria)
*   [ ] クエストクリア時にキャラクター獲得演出（宝箱オープン等）が表示されること。
*   [ ] 図鑑画面で取得済みキャラの詳細（画像、名前）が確認できること。
*   [ ] 未取得キャラがシルエット等の状態で表示され、総数がわかること。
*   [ ] 進化条件を満たしたキャラに対し「進化」アクションができ、画像とステータスが変化すること。
