# Epic: ゲーム開始と継続性 (Login & Onboarding)

## 概要
子供がアプリを開いた瞬間から、ストレスなく「自分の世界」に入り込めるようにする。複雑な認証を排除しつつ、継続プレイのデータを確実に保持する。

## ユーザーストーリー

### 1. 名前入力（冒険者登録）
*   **As a (誰):** 子供
*   **I want to (何):** アプリを初回起動した時に、「あなたのなまえをおしえてね」と優しく聞かれ、自分の名前（ひらがな）を入力したい。
*   **So that (なぜ):** これから始まるのが、他の誰でもない「私だけの物語」であることを認識し、愛着を持ちたいから。

### 2. ストレスフリーな再開（オートログイン）
*   **As a (誰):** 子供
*   **I want to (何):** 2回目以降にアプリを開いた時、IDやパスワードを求められることなく、「おかえり、よりちゃん！」と即座に前回の続きから始めたい。
*   **So that (なぜ):** 「やりたい！」と思った瞬間の熱量を逃さず、0秒でゲームの世界に没入したいから。

### 3. データの自動保存
*   **As a (誰):** 子供
*   **I want to (何):** 図鑑の状況や、育てたキャラのデータが、特別な「セーブ操作」をしなくても勝手に保存されていてほしい。
*   **So that (なぜ):** 「セーブし忘れてデータが消えた」という悲劇を回避し、安心して遊び続けたいから。

## 完了基準 (Acceptance Criteria)
*   [ ] 初回起動時に名前入力フォームが表示され、入力した名前が保存されること。
*   [ ] 入力された名前が、ホーム画面やメッセージ等で表示されること（「ようこそ 〇〇ちゃん」）。
*   [ ] 2回目以降の起動時、名前入力がスキップされ、保存されたデータがロードされた状態でホーム画面が表示されること。
*   [ ] ゲームの進行状況（クリア数、所持キャラ、エサ数）がLocalStorage等に永続化されること。
