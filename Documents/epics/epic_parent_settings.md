# Epic: 保護者設定とご褒美システム

## 概要
デジタルなゲーム内の達成感を、リアルな親子のコミュニケーションや報酬に接続するための機能。保護者が「応援（サポート）」をシステム的に組み込めるようにする。

## ユーザーストーリー

### 1. 保護者メニューへの安全なアクセス
*   **As a (誰):** 親
*   **I want to (何):** 子供には解除しにくい特定の方法（例：長押し、「3+5は？」のような計算問題）でのみ、「保護者メニュー」を開けるようにしたい。
*   **So that (なぜ):** ご褒美の設定や学習履歴などの管理画面を、子供の誤操作から守りつつ、必要な時にすぐにアクセスしたいから。

### 2. カスタムご褒美（クエスト）の作成
*   **As a (誰):** 親
*   **I want to (何):** 「サイゼリヤに行く」「新しい本を買う」など、自由なテキストでご褒美を作成し、達成条件（キャラ20体ゲットなど）と紐付けたい。
*   **So that (なぜ):** 汎用的な報酬ではなく、我が子に一番響く「オーダーメイドの目標」を設定して、やる気を最大化させたいから。

### 3. ご褒美チケットのサプライズ通知
*   **As a (誰):** 子供
*   **I want to (何):** 条件を達成した瞬間に、画面上でファンファーレが鳴り、「パパ・ママからのプレゼント！」としてご褒美チケットが表示されてほしい。
*   **So that (なぜ):** 自分の頑張りが、デジタルの外にある「リアルな幸せ」に繋がったことを実感し、親への感謝と達成感を同時に味わいたいから。

### 4. チケットの管理と消し込み
*   **As a (誰):** 親
*   **I want to (何):** 子供が獲得したチケットを一覧で確認し、実際にサイゼリヤに行った後などで「使用済み」ステータスに変更したい。
*   **So that (なぜ):** 「ご褒美あげたっけ？」という曖昧さをなくし、子供との約束（契約）を確実に履行することで信頼関係を深めたいから。

## 完了基準 (Acceptance Criteria)
*   [ ] 保護者メニューへのゲート（計算問題等の認証）が実装されていること。
*   [ ] ご褒美名（テキスト）と達成条件（プルダウン選択）を入力・保存できること。
*   [ ] プレイ中に条件が達成された際、モーダル等でご褒美内容が通知されること。
*   [ ] 獲得したご褒美を一覧表示し、使用済み（Done）にステータス変更できること。
